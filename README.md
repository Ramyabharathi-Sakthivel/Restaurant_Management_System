RESTAURANT MANAGEMENT SYSTEM :

A simple Restaurant Management System developed using Java, Servlets, JDBC, and Oracle/MySQL Database.
This project helps manage restaurant order records such as customer details, food items, quantity, billing, and order history through a web interface.

PROJECT OVERVIEW :

The Restaurant Management System allows administrators to add, view, and manage customer food orders efficiently. It stores all order details in a database and retrieves them when required, ensuring organized record keeping and quick access to information.

FEATURES :

-- Add Record – Insert new customer order details into the database.

-- View Record – Retrieve a specific order using customer name and order date.

-- View All Records – Display all stored restaurant orders.

-- Duplicate Check – Prevents insertion of already existing records.

-- Auto Record ID Generation – Generates unique order IDs using sequence and date.

-- Input Validation – Ensures valid customer name and required fields before storing data.

TECHNOLOGIES USED :

-- Java

-- Servlets

-- JDBC

-- HTML 

-- Apache Tomcat

-- Oracle / MySQL Database

MODULES :

Administrator Service – Handles validation and business logic.

DAO Layer – Manages database operations such as insert, fetch, and list.

Bean Classes – Stores restaurant order data.

Utility Classes – Database connection and custom exceptions.

PURPOSE :

This project is designed to simplify restaurant order management by maintaining accurate records and reducing manual effort.
It is suitable for learning Java Web Development concepts like MVC architecture, JDBC connectivity, and servlet handling.
